BaseLots Smart Contracts

Fractional real estate security token infrastructure

BLOCKS Model: Each property has its own token (e.g., BLOCKS-Austin-001, BLOCKS-Phoenix-001). There is no single "BaseLots Token". Investors purchase BLOCKS specific to each property.

📄 BaseLotsToken.sol

ERC-3643 security token contract representing BLOCKS (fractional shares of a specific property). Each property deployment creates a new token instance (e.g., BLOCKS-Austin-001). Handles minting during property offering, compliance-enforced transfers, and dividend distribution per property.

🆔 IdentityRegistry.sol

On-chain KYC/AML registry shared across all properties. Links wallet addresses to verified identities, tracks accreditation status, and supports multi-wallet linking per identity. One registration works for all BLOCKS properties.

✓ Compliance.sol

Modular compliance engine enforcing transfer rules per property: max balance limits, holding periods, Reg CF investor caps (500 per property), and minimum investments.

🛡️ HeritageShield.sol

Automated inheritance protocol (HSP) that works across all BLOCKS holdings. Beneficiary designation aggregates across all properties, 90-day challenge period, oracle integration for death verification, and trustless execution.
